# Вскроем все — лендинг

Одностраничный сайт услуги вскрытия и ремонта замков по Москве и Подмосковью.

- **Прод:** [https://вскроемвсе.рф](https://вскроемвсе.рф)
- **Хостинг:** Vercel (статика + serverless-функция приёма заявок)
- **Домен:** REG.RU, привязан к Vercel через DNS

## Структура

- `public/index.html` — главная страница
- `public/assets/` — изображения, robots, sitemap
- `public/src/css/` — стили (tokens, base, layout, components, utilities)
- `public/src/js/` — клиентский код (config, форма, модалка, инициализация)
- `api/mock.js` — Vercel-функция `POST /api/mock` (валидация, антиспам, Telegram)
- `lib/leadAntispam.js` — общая логика лимитов (honeypot, время, счётчики IP/телефона)
- `server.js` — локальный Node-сервер для разработки
- `vercel.json` — конфиг Vercel
- `.env.example` — шаблон переменных окружения

## Локальный запуск

Нужен Node.js 18+:

```bash
npm start
```

Откройте: [http://localhost:8000](http://localhost:8000)

## Антиспам и лимиты заявок

На клиенте: скрытое поле-ловушка для ботов, минимум ~2,5 с между загрузкой страницы и отправкой, пауза **45 с** между повторными отправками с одного браузера (`localStorage`).

На сервере: не более **3 заявок за 24 часа** отдельно по **IP** и отдельно по **нормализованному телефону**. При **4-й** попытке в том же 24-часовом окне — **блокировка на 24 ч** для затронутого IP или номера (ответ `429`).

На **Vercel** счётчики хранятся в **Redis / KV** — подключите интеграцию Storage в проекте и задайте переменные:

| Переменная          | Назначение                                     |
| ------------------- | ---------------------------------------------- |
| `KV_REST_API_URL`   | URL REST API Redis (из интеграции Vercel / KV) |
| `KV_REST_API_TOKEN` | токен доступа                                  |

Без этих переменных на проде работают только клиентские проверки и honeypot; **серверный лимит 3/24 ч не действует** (в логах функции будет предупреждение). Локально `npm start` использует in-memory счётчики, если KV не задан.

## Заявки и Telegram-бот

Форма отправляет `POST /api/mock`. После валидации заявка летит в Telegram-бота.

Переменные окружения (в Vercel → Settings → Environment Variables, либо в локальном `.env`):

| Переменная           | Назначение                                           |
| -------------------- | ---------------------------------------------------- |
| `TELEGRAM_ENABLED`   | `true` — включить отправку в Telegram                |
| `TELEGRAM_BOT_TOKEN` | токен бота от `@BotFather`                           |
| `TELEGRAM_CHAT_ID`   | id чата/лички, куда падают заявки                    |
| `SITE_URL`           | URL сайта для подписи источника в уведомлении        |

Если переменные не заданы или Telegram недоступен, форма всё равно вернёт пользователю успех — заявку видно в логах функции (Vercel → Deployments → Functions → Logs).

## Как поменять контент

| Что                    | Где                                                       |
| ---------------------- | --------------------------------------------------------- |
| Телефон                | `public/src/js/config.js` → `PHONE_DISPLAY`, `PHONE_TEL`  |
| Ссылка VK              | `public/src/js/config.js` → `VK_URL`                      |
| Города/география       | `public/src/js/config.js` → `SERVICE_AREAS`               |
| Тексты разделов        | `public/index.html`                                       |
| Главное фото (hero)    | `public/assets/images/hero-main.png`                      |
| Галерея                | `public/assets/images/site/` (подключённые в `index.html` файлы) |
| Юридический текст      | блок `#policy` в `public/index.html`                      |

## Деплой

Любой пуш в ветку `main` запускает автодеплой на Vercel.
